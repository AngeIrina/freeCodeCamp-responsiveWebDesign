Introduction

GitHub is a code hosting platform that enables version control and collaboration. Allows you and others to collaborate on projects from anywhere.

Collaborative work, one of the fundamental features of GitHub, is made possible by the open source software Git, on which GitHub is based.

Key benefits of GitHub:

  • Open source communities
  • Version control
  • Project portfolio
  • Team collaboration
About Git

Git is a version control system that keeps track of changes to files. It's especially useful when you and a group of people are making changes to the same files at the same time.

Typically, to do this in a Git-based workflow, you would do the following:

  • Create a branch from the main copy of files you (and your collaborators) are working on.
  • Make changes to files independently and securely in your own personal branch.
  • Dejar que Git fusione mediante combinación y de forma inteligente los cambios específicos en la copia principal de archivos, de modo que los cambios no afecten a las actualizaciones de otras personas.
  • Let Git keep track of your changes and those of other people, so everyone keeps working on the most up-to-date version of the project.
About GitHub

GitHub is a platform where you can store, share, and work together with other users to write code.

Storing your code in a "repository" on GitHub allows you to:

  • Present or share work.
  • Track and manage code changes over time.
  • Let other users review the code and make suggestions to improve it.
  • Collaborate on a shared project, without worrying that changes will affect collaborators' work before you are ready to integrate them.
How do Git and GitHub work together?

When you upload files to GitHub, you store them in a "Git repository." This means that when you make changes (or "commits") to files on GitHub, Git will automatically start to track and manage the changes. There are many Git-related actions you can complete on GitHub right in the browser, such as creating a Git repository, creating branches, and uploading and editing files. However, most people work on their files locally (on their own computer), then continually sync these local changes, and all related Git data, to the "remote" central repository on GitHub. There are many tools you can use to do this, such as GitHub Desktop.

When you start collaborating with others and you all need to work on the same repository at the same time, you will continually do the following:

  • Pull all the most recent changes made by collaborators from the remote repository on GitHub.
  • Push changes back to the same remote repository on GitHub.
Creating a GitHub account

To get started with GitHub, you'll need to create a free personal account on GitHub.com and verify your email address.

Each person using GitHub.com will need to log in to a personal account. Your personal account is your identity on GitHub.com and has a username and profile.
